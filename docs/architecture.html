MiniSAM Architecture

MiniSAM is a small SAM-style promptable segmentation model built from scratch. It follows the same high-level idea as Segment Anything: encode the image, encode the prompt, and use a mask decoder to predict the requested object mask.

1. Image Encoder

The image encoder converts the input image into a dense feature map. In MiniSAM, the input image is resized to 256 × 256 and divided into 16 × 16 patches. 

Input image:       [B, 3, 256, 256]
Patch embedding:   [B, 256, 16, 16]
Flattened tokens:  [B, 256, C]
Transformer blocks
Output embedding:  [B, 256, 16, 16]

The image encoder uses ViT-style self-attention. This allows image patches to exchange information and learn object shape, texture, and boundary cues.

2. Prompt Encoder

The prompt encoder converts user prompts into embeddings. MiniSAM supports point prompts, box prompts, and optional mask prompts.

Point Prompt

Point coordinate: [x, y]
Point label:      positive or negative
Output:           sparse prompt embedding

Box Prompt

Box: [x1, y1, x2, y2]
Top-left corner embedding
Bottom-right corner embedding
Output: sparse prompt embedding

Mask Prompt

Previous mask: [B, 1, 256, 256]
Small CNN downsampling
Output: dense prompt embedding [B, 256, 16, 16]

For our Oxford Pet training, we generate a box prompt and one positive point prompt from the ground-truth mask.

3. Mask Decoder

The mask decoder is the most important part of MiniSAM. It decides which pixels satisfy the prompt. It receives image embeddings, prompt embeddings, learned mask tokens, and an IoU token. 

Inputs:
Image embedding:          [B, 256, 16, 16]
Sparse prompt embedding:  [B, N, 256]
Dense prompt embedding:   [B, 256, 16, 16]
Mask tokens:              learned output tokens
IoU token:                learned quality token

4. Two-Way Attention

MiniSAM uses a two-way transformer inside the mask decoder. The key idea is that prompt tokens attend to image tokens, and image tokens also attend back to prompt tokens. 

Step 1: Prompt tokens use self-attention
Step 2: Prompt tokens attend to image tokens
Step 3: Prompt tokens are updated by an MLP
Step 4: Image tokens attend back to prompt tokens
Step 5: Updated tokens are used for mask prediction

This is how the model connects the request from the prompt with the visual information in the image.

5. Hypernetwork Mask Head

After two-way attention, each mask token is converted into a small dynamic classifier using an MLP. This classifier scores every pixel feature. 

Mask token
   ↓
MLP hypernetwork
   ↓
Dynamic classifier vector
   ↓
Dot product with each pixel feature
   ↓
Mask logits

A pixel receives a high score when its feature matches the prompt-conditioned mask token. After thresholding, high-score pixels become foreground.

6. Loss Function

MiniSAM uses a SAM-style best-mask training objective. Since the decoder predicts multiple candidate masks, we select the best candidate per sample and train that mask strongly. 

Total loss = selected mask loss + IoU prediction loss

selected mask loss =
    20 × focal loss
    + 1 × dice loss

IoU prediction loss =
    MSE(predicted IoU, actual IoU)

The final model uses the corrected SAM-style loss. This fixed the earlier validation-loss explosion caused by averaging loss across all mask candidates.

7. Final Architecture Summary

Component Role
Image Encoder Converts image into visual feature map using ViT-style self-attention.
Prompt Encoder Converts points, boxes, and masks into prompt embeddings.
Two-Way Transformer Allows prompt tokens and image tokens to exchange information.
Mask Tokens Represent candidate segmentation masks.
Hypernetwork Head Turns each mask token into a dynamic pixel classifier.
IoU Head Predicts mask quality and helps select the best mask.
