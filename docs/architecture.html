How MiniSAM Works

MiniSAM follows a simple idea: combine what the model sees in the image with what the user asks through a prompt. The result is a segmentation mask.

First, understand the output: what is a mask?

A mask is an image-sized map. Each pixel answers one question: does this pixel belong to the object?

White pixel

The pixel belongs to the object.

Black pixel

The pixel belongs to the background.

Goal of MiniSAM: produce a clean mask for the object selected by the user's prompt.

Architecture Diagram: Task, Model, and Data

This diagram explains the main idea behind SAM in three parts: what task SAM solves, how the model is built, and what data was used to train it.

SAM has three important ideas: a user gives a prompt, the model predicts a valid mask, and large-scale data is used to train the model.

(a) Task: Promptable Segmentation

SAM is designed to segment an object using different types of prompts. A prompt is a simple instruction given by the user.

  • Point prompt: Click on the object you want.
  • Box prompt: Draw a rectangle around the object.
  • Mask prompt: Give a rough mask as guidance.
  • Text prompt: Describe the object using words.

The image and the prompt are given to the model, and the model returns a valid mask, which means a clean object region.

(b) Model: Segment Anything Model

SAM uses three main components to convert an image and a prompt into a mask.

  • Image encoder: Reads the image and creates useful image features.
  • Prompt encoder: Converts the user prompt, such as a point or box, into prompt features.
  • Lightweight mask decoder: Combines image features and prompt features to predict the final segmentation mask.

In simple words, the image encoder understands the image, the prompt encoder understands the user's instruction, and the mask decoder produces the object mask.

(c) Data: Data Engine and SA-1B Dataset

SAM was trained using a large data engine. The model helped create masks, humans checked or improved them, and the improved data was used to train the model again.

This repeated cycle created the SA-1B dataset, which contains:

  • 1+ billion masks
  • 11 million images
  • Privacy-respecting and licensed images

This large dataset is one reason SAM can work on many different kinds of images and objects.

Part 1: Image Encoder

The image encoder is the part that reads the image. It does not directly output the final mask. Instead, it creates image features: a compressed but meaningful representation of the image.

Beginner analogy

Think of the image encoder like a person describing a photo. It notices where the object is, where the edges are, what textures are present, and how the background looks.

What happens inside?

  1. The image is resized to a fixed size.
  2. The image is split into smaller patches.
  3. The model converts those patches into feature vectors.
  4. The feature vectors are refined so each region has context from the image.

Part 2: Prompt Encoder

The prompt encoder reads the user's instruction. A prompt is important because it tells the model what to segment. Without a prompt, the model may not know which object the user wants.

Positive point

A click inside the target object. It means: this region should be included.

Negative point

A click on background or another object. It means: this region should be excluded.

Box

A rectangle around the object. It gives the model a search area.

How are prompts represented?

A click has coordinates such as (x, y). A box has four coordinates such as (x1, y1, x2, y2). The prompt encoder converts these numbers into prompt features, so the decoder can use them together with image features.

Part 3: Mask Decoder

The mask decoder is the main decision-making part. It receives image features and prompt features, then predicts one or more masks.

Beginner analogy

Imagine a detective. The image features are the scene. The prompt features are the clue. The decoder uses both to decide which pixels belong to the selected object.

Why does the decoder need both?

  • The image features tell the model what is visible.
  • The prompt features tell the model what the user wants.
  • The decoder connects both sources of information.

Part 4: IoU Predictor

The model can produce multiple mask candidates. The IoU predictor estimates which candidate is best.

IoU means Intersection over Union. In simple words, it measures how much the predicted mask overlaps the correct object mask.

IoU value Meaning
Close to 0 The predicted mask barely matches the real object.
Around 0.5 The mask is partly correct but still has many mistakes.
Close to 1 The predicted mask matches the real object very well.
Output: quality scores for the candidate masks.

Complete flow with one example

Let us say the image contains a cat and the user clicks on the cat.

  1. Image encoder: reads the image and creates image features.
  2. Prompt encoder: reads the click location and creates prompt features.
  3. Mask decoder: combines both and predicts possible cat masks.
  4. IoU predictor: scores the masks and selects the best one.
  5. Final output: a binary mask showing the cat.

Why multiple masks?

A prompt can be ambiguous. If you click on a pet's face, the correct answer could be the face, the head, or the whole animal. Producing multiple masks helps the model handle this uncertainty.

How MiniSAM is different from a normal segmentation model

Normal segmentation MiniSAM-style segmentation
Usually predicts a fixed class, such as cat or dog. Predicts the object selected by the user prompt.
The same image usually gives the same output. The same image can give different outputs for different prompts.
Less interactive. Interactive: click or box guides the model.

Key takeaways

  • The image encoder understands the photo.
  • The prompt encoder understands the user instruction.
  • The mask decoder combines image plus prompt to predict masks.
  • The IoU predictor chooses the best mask candidate.
  • The prompt is what makes the model interactive.
