MiniSAM Architecture

MiniSAM is a small SAM-style promptable segmentation model built from scratch. It follows the same high-level idea as Segment Anything: encode the image, encode the prompt, and use a mask decoder to predict the requested object mask.

Input Image
→
Image Encoder
→
Image Embedding
+
Prompt Embedding
→
Mask Decoder
→
Final Mask

1. Image Encoder

The image encoder converts the input image into a dense feature map. In MiniSAM, the input image is resized to 256 × 256 and divided into 16 × 16 patches. 

Input image:       [B, 3, 256, 256]
Patch embedding:   [B, 256, 16, 16]
Flattened tokens:  [B, 256, C]
Transformer blocks
Output embedding:  [B, 256, 16, 16]

The image encoder uses ViT-style self-attention. This allows image patches to exchange information and learn object shape, texture, and boundary cues.

Image
→
Patch Embedding
→
Transformer Blocks
→
Image Feature Map

2. Prompt Encoder

The prompt encoder converts user prompts into embeddings. MiniSAM supports point prompts, box prompts, and optional mask prompts.

Point Prompt

Point coordinate: [x, y]
Point label:      positive or negative
Output:           sparse prompt embedding

Box Prompt

Box: [x1, y1, x2, y2]
Top-left corner embedding
Bottom-right corner embedding
Output: sparse prompt embedding

Mask Prompt

Previous mask: [B, 1, 256, 256]
Small CNN downsampling
Output: dense prompt embedding [B, 256, 16, 16]

For Oxford-IIIT Pet training, MiniSAM generates a box prompt and one positive point prompt from the ground-truth mask.

3. Mask Decoder

The mask decoder is the most important part of MiniSAM. It decides which pixels satisfy the prompt. It receives image embeddings, prompt embeddings, learned mask tokens, and an IoU token. 

Inputs:
Image embedding:          [B, 256, 16, 16]
Sparse prompt embedding:  [B, N, 256]
Dense prompt embedding:   [B, 256, 16, 16]
Mask tokens:              learned output tokens
IoU token:                learned quality token

4. Two-Way Attention

MiniSAM uses a two-way transformer inside the mask decoder. The key idea is that prompt tokens attend to image tokens, and image tokens also attend back to prompt tokens. 

Step 1: Prompt tokens use self-attention
Step 2: Prompt tokens attend to image tokens
Step 3: Prompt tokens are updated by an MLP
Step 4: Image tokens attend back to prompt tokens
Step 5: Updated tokens are used for mask prediction

This is how the model connects the request from the prompt with the visual information in the image.

5. Hypernetwork Mask Head

After two-way attention, each mask token is converted into a small dynamic classifier using an MLP. This classifier scores every pixel feature. 

Mask token
   ↓
MLP hypernetwork
   ↓
Dynamic classifier vector
   ↓
Dot product with each pixel feature
   ↓
Mask logits

A pixel receives a high score when its feature matches the prompt-conditioned mask token. After thresholding, high-score pixels become foreground.

6. Loss Function

MiniSAM uses a SAM-style best-mask training objective. Since the decoder predicts multiple candidate masks, we select the best candidate per sample and train that mask strongly. 

Total loss = selected mask loss + IoU prediction loss

selected mask loss =
    20 × focal loss
    + 1 × dice loss

IoU prediction loss =
    MSE(predicted IoU, actual IoU)

The final model uses the corrected SAM-style loss. This fixed the earlier validation-loss explosion caused by averaging loss across all mask candidates.

Tensor Shape Flow

Tensor shapes are one of the most important parts of understanding a SAM-style model. The table below shows how the image, prompts, tokens, and masks move through MiniSAM.

Stage Tensor Shape Meaning
Input image [B, 3, 256, 256] Batch of RGB images.
Patch embedding [B, 256, 16, 16] Image converted into a low-resolution dense feature map.
Image tokens [B, 256, 256] There are 256 patch tokens, each with 256 channels.
Point prompt [B, 1, 2] One point coordinate: [x, y].
Point label [B, 1] Positive point label. In this project, label 1 means foreground.
Box prompt [B, 4] Bounding box: [x1, y1, x2, y2].
Sparse prompt embedding [B, 3, 256] One point token plus two box-corner tokens.
Dense prompt embedding [B, 256, 16, 16] Mask/no-mask embedding aligned with image embedding resolution.
Mask decoder output [B, 3, 256, 256] Three candidate masks predicted for each image.
IoU prediction output [B, 3] Predicted quality score for each candidate mask.

In short, MiniSAM first compresses the image into a 16 × 16 feature grid, combines it with prompt embeddings, and then upsamples the decoder output back to a 256 × 256 mask.

Why Does MiniSAM Predict Multiple Masks?

A prompt can be ambiguous. For example, if a user clicks on an animal's body, the correct mask could mean the full animal, only the head, only the torso, or the animal plus some boundary region. To handle this ambiguity, MiniSAM predicts multiple candidate masks instead of only one.

One prompt
→
Mask candidate 1
Mask candidate 2
Mask candidate 3
→
IoU head selects best mask

In this project, the mask decoder predicts three candidate masks: 

Predicted masks: [B, 3, 256, 256]
Predicted IoUs:  [B, 3]

The IoU prediction head gives a quality score for each mask. During inference, MiniSAM selects the mask with the highest predicted IoU score. 

selected_idx = torch.argmax(pred_ious, dim=1)

selected_mask = pred_masks[
    torch.arange(pred_masks.shape[0]),
    selected_idx,
]

This worked well in the final experiment. The realistic selected-mask IoU was very close to the oracle best-mask IoU:

Selection Method Validation IoU
Selected by predicted IoU head 0.8949
Oracle best mask among 3 candidates 0.8954

The small gap between 0.8949 and 0.8954 shows that the IoU head learned to choose almost the same mask as the oracle best candidate.

MiniSAM vs Original SAM

MiniSAM is not a full reproduction of the original Segment Anything Model. Instead, it is a small educational implementation that keeps the main architectural ideas of SAM while making the model small enough to train from scratch on a small dataset.

Component Original SAM MiniSAM
Goal General-purpose promptable segmentation Educational promptable segmentation from scratch
Image size Large image resolution, commonly 1024 × 1024 256 × 256
Image encoder Large ViT encoder such as ViT-B, ViT-L, or ViT-H Small ViT-style encoder
Dataset scale Very large-scale segmentation data Oxford-IIIT Pet segmentation dataset
Prompt types Point, box, and mask prompts Point, box, and optional mask prompts
Mask decoder Two-way transformer decoder Small two-way transformer decoder
Mask outputs Multiple candidate masks plus IoU scores Three candidate masks plus IoU scores
Losses Mask loss and IoU prediction loss Focal loss, dice loss, and IoU prediction loss
Purpose Strong zero-shot segmentation system Learning how SAM works internally

The important point is that MiniSAM keeps the key mechanism:

\[ \text{Image Embedding} + \text{Prompt Embedding} \rightarrow \text{Prompt-Conditioned Mask} \]

This makes MiniSAM useful for understanding how a promptable segmentation model works, even though it is much smaller than the original SAM.

7. Final Architecture Summary

Component Role
Image Encoder Converts image into visual feature map using ViT-style self-attention.
Prompt Encoder Converts points, boxes, and masks into prompt embeddings.
Two-Way Transformer Allows prompt tokens and image tokens to exchange information.
Mask Tokens Represent candidate segmentation masks.
Hypernetwork Head Turns each mask token into a dynamic pixel classifier.
IoU Head Predicts mask quality and helps select the best mask.
