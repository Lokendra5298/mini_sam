Training Details

MiniSAM is trained as a promptable segmentation model. Instead of directly learning image → mask, it learns:

\[ \text{Image} + \text{Prompt} \rightarrow \text{Segmentation Mask} \]

During training, each ground-truth mask is used to generate a box prompt and a positive point prompt. The model then predicts multiple candidate masks and learns to select the best one using an IoU prediction head. 

python train.py \
  --data_root data/oxford_pet/processed \
  --out_dir runs/minisam_oxford_pet_v2_sam_loss_ep50_bs32 \
  --device_id 1 \
  --epochs 50 \
  --batch_size 32 \
  --workers 8 \
  --lr 1e-4

Dataset and Prompt Generation

The model is trained on the Oxford-IIIT Pet segmentation dataset. Each sample contains an RGB image and a binary foreground mask.

\[ x \in \mathbb{R}^{3 \times 256 \times 256}, \qquad y \in \{0,1\}^{1 \times 256 \times 256} \]

From the ground-truth mask, MiniSAM generates:

  • A box prompt: [x1, y1, x2, y2]
  • A positive point prompt sampled from the object region
ys, xs = np.where(mask_np > 0)

1. Image Encoder

The image encoder converts the input image into dense visual tokens using a ViT-style patch encoder. The input image is divided into non-overlapping patches using a convolution with kernel size and stride equal to the patch size.

\[ N = \frac{H}{P} \times \frac{W}{P} \]

For MiniSAM:

\[ H = W = 256, \quad P = 16, \quad N = 16 \times 16 = 256 \]

The patch embedding is:

\[ z_0 = \text{Flatten}(\text{Conv}_{P \times P, stride=P}(x)) + E_{pos} \]

Then transformer blocks update the visual tokens:

\[ z'_l = z_{l-1} + \text{MSA}(\text{LN}(z_{l-1})) \] \[ z_l = z'_l + \text{MLP}(\text{LN}(z'_l)) \] 
self.patch_embed = nn.Conv2d(
    in_channels,
    embed_dim,
    kernel_size=patch_size,
    stride=patch_size,
)

self.pos_embed = nn.Parameter(
    torch.zeros(1, num_patches, embed_dim)
)

x = self.patch_embed(image)          # [B, C, 16, 16]
x = x.flatten(2).transpose(1, 2)     # [B, 256, C]
x = x + self.pos_embed

for block in self.blocks:
    x = block(x)

Final image encoder output: 

Image embedding: [B, 256, 16, 16]

2. Prompt Encoder

The prompt encoder converts point, box, and mask prompts into sparse and dense prompt embeddings. Point and box prompts become sparse tokens. Mask prompts become dense feature maps.

Point Prompt Formula

A point coordinate is first normalized to the range [0, 1]:

\[ p = \left(\frac{x}{W}, \frac{y}{H}\right) \]

MiniSAM uses random Fourier positional features:

\[ \phi(p) = \left[ \sin(2\pi (2p-1)G), \cos(2\pi (2p-1)G) \right] \]

Then a prompt-type embedding is added:

\[ e_{point} = \phi(p) + e_{type} \] 
coords = coords / image_size
coords = 2.0 * coords - 1.0
coords = coords @ gaussian_matrix
coords = 2.0 * torch.pi * coords

pos_embed = torch.cat(
    [torch.sin(coords), torch.cos(coords)],
    dim=-1,
)

point_embedding = pos_embed + point_type_embedding

Box Prompt Formula

A box is represented using two corner points:

\[ b = [x_1, y_1, x_2, y_2] \]
\[ e_{box} = [\phi(x_1,y_1) + e_{top-left}, \phi(x_2,y_2) + e_{bottom-right}] \] 
corners = torch.stack([
    boxes[:, 0:2],
    boxes[:, 2:4],
], dim=1)

corner_embedding = self.pe_layer(corners)
corner_embedding[:, 0, :] += top_left_embedding
corner_embedding[:, 1, :] += bottom_right_embedding

Mask Prompt

A previous mask prompt is downsampled using a small CNN so that it matches the image embedding resolution. 

mask_embedding = self.mask_downscaling(mask)

# Output:
# [B, 256, 16, 16]

3. Mask Decoder

The mask decoder uses two-way attention to connect prompt tokens with image tokens. This is the part that decides which pixels satisfy the prompt request.

First, MiniSAM creates learned output tokens: 

IoU token
Mask token 1
Mask token 2
Mask token 3
Prompt tokens

The attention operation is:

\[ \text{Attention}(Q,K,V) = \text{softmax} \left( \frac{QK^T}{\sqrt{d}} \right)V \]

The two-way transformer performs:

\[ T \leftarrow T + \text{Attn}(T,T,T) \] \[ T \leftarrow T + \text{Attn}(T,I,I) \] \[ I \leftarrow I + \text{Attn}(I,T,T) \]

Here, T represents prompt/output tokens and I represents image tokens. 

# Token self-attention
tokens = tokens + self.token_self_attn(tokens, tokens, tokens)

# Prompt tokens attend to image tokens
tokens = tokens + self.token_to_image_attn(
    tokens,
    image_tokens,
    image_tokens,
)

# Image tokens attend back to prompt tokens
image_tokens = image_tokens + self.image_to_token_attn(
    image_tokens,
    tokens,
    tokens,
)

Hypernetwork Mask Head

Each mask token becomes a dynamic pixel classifier.

\[ h_j = \text{MLP}(t_j) \]

The mask logit for pixel i is computed by a dot product:

\[ m_{j,i} = h_j^T f_i \]

where h_j is the classifier from mask token j and f_i is the upscaled pixel feature. 

mask_token = mask_tokens_out[:, i, :]
hyper_vector = self.output_hypernetworks_mlps[i](mask_token)

# upscaled_embedding: [B, C, H, W]
features = upscaled_embedding.view(B, C, H * W)

mask_logits = torch.matmul(hyper_vector, features)
mask_logits = mask_logits.view(B, 1, H, W)

4. Losses

MiniSAM uses focal loss, dice loss, and IoU prediction loss. The corrected version uses a SAM-style best-mask objective: among multiple predicted masks, the best mask candidate is selected for the main segmentation loss.

Focal Loss

Focal loss handles foreground-background imbalance.

\[ \text{FL}(p,y) = -\alpha y(1-p)^\gamma \log(p) - (1-\alpha)(1-y)p^\gamma \log(1-p) \] 
prob = inputs.sigmoid()

ce_loss = F.binary_cross_entropy_with_logits(
    inputs,
    targets,
    reduction="none",
)

p_t = prob * targets + (1.0 - prob) * (1.0 - targets)
loss = ce_loss * ((1.0 - p_t) ** gamma)

Dice Loss

Dice loss measures mask overlap.

\[ \text{DiceLoss} = 1 - \frac{2\sum_i p_i y_i + \epsilon} {\sum_i p_i + \sum_i y_i + \epsilon} \] 
inputs = inputs.sigmoid()

numerator = 2.0 * (inputs * targets).sum(dim=1)
denominator = inputs.sum(dim=1) + targets.sum(dim=1)

dice_loss = 1.0 - (numerator + eps) / (denominator + eps)

IoU Prediction Loss

The IoU head predicts the quality of each candidate mask.

\[ \text{IoU} = \frac{|M \cap Y|} {|M \cup Y|} \]
\[ \mathcal{L}_{iou} = \text{MSE} (\hat{q}, \text{IoU}(M,Y)) \]

Final SAM-Style Loss

For each predicted mask j, MiniSAM computes:

\[ \mathcal{L}_{mask}^{(j)} = 20 \cdot \mathcal{L}_{focal}^{(j)} + \mathcal{L}_{dice}^{(j)} \]

Then the best mask is selected:

\[ j^* = \arg\min_j \mathcal{L}_{mask}^{(j)} \]

Final loss:

\[ \mathcal{L} = \mathcal{L}_{mask}^{(j^*)} + \mathcal{L}_{iou} \] 
best_idx = torch.argmin(all_mask_losses.detach(), dim=1)

selected_mask_loss = all_mask_losses[
    batch_indices,
    best_idx,
].mean()

iou_loss = F.mse_loss(
    pred_ious,
    all_actual_ious.detach(),
)

total_loss = selected_mask_loss + iou_loss

Old Loss vs Corrected SAM-Style Loss

During the first training attempt, the loss was averaged over all predicted mask candidates. This created a problem: even if one candidate mask was very good, the other weaker candidate masks still increased the validation loss.

First Attempt: Average Loss Over All Masks

The first version used:

\[ \mathcal{L}_{avg} = \frac{1}{M} \sum_{j=1}^{M} \mathcal{L}_{mask}^{(j)} \]

where M is the number of predicted mask candidates. In MiniSAM, M = 3. 

total_focal = 0.0
total_dice = 0.0

for i in range(num_masks):
    mask_i = pred_masks[:, i:i + 1]
    total_focal += focal_loss(mask_i, gt_masks)
    total_dice += dice_loss(mask_i, gt_masks)

total_loss = total_focal / num_masks + total_dice / num_masks

This made training unstable because the model was punished for all mask candidates, even when one candidate already matched the object well.

Observed Problem

In the first 120-epoch run, the training loss kept decreasing, but the validation loss increased strongly after the early epochs. This indicated overfitting and also showed that averaging over all mask candidates was not ideal for a SAM-style decoder.

Corrected Approach: Best-Mask Training

The corrected version follows a SAM-style idea. For each image, MiniSAM first computes the loss for every candidate mask. Then it chooses the best mask candidate and backpropagates the segmentation loss through that selected mask.

\[ j^* = \arg\min_j \mathcal{L}_{mask}^{(j)} \]
\[ \mathcal{L} = \mathcal{L}_{mask}^{(j^*)} + \mathcal{L}_{iou} \] 
all_mask_losses = torch.stack(all_mask_losses, dim=1)

best_idx = torch.argmin(
    all_mask_losses.detach(),
    dim=1,
)

selected_mask_loss = all_mask_losses[
    batch_indices,
    best_idx,
].mean()

iou_loss = F.mse_loss(
    pred_ious,
    all_actual_ious.detach(),
)

total_loss = selected_mask_loss + iou_loss

Why This Is Better

A prompt can be ambiguous, so multiple mask candidates are useful. The model should not be forced to make every candidate mask identical. Instead, one candidate should match the target well, and the IoU head should learn which candidate is best.

Training Version Validation IoU Validation Dice Observation
Old averaged multi-mask loss 0.8677 0.9285 Validation loss increased strongly.
Corrected SAM-style best-mask loss 0.8949 0.9440 Better IoU and cleaner training behavior.

This was one of the most important lessons in the project: for a SAM-style multi-mask decoder, the training objective should respect mask ambiguity.

Common Mistakes When Implementing SAM From Scratch

Building a SAM-style model from scratch is not only about writing the modules. Many implementation details matter. Below are the main mistakes that can make training unstable or confusing.

Mistake Why It Is a Problem Better Approach
Training only image → mask SAM is not a normal segmentation model. It is a prompt-conditioned model. Train using image + prompt → mask.
Forgetting coordinate normalization Point and box prompts are pixel coordinates, but the prompt encoder needs normalized coordinates. Convert coordinates to the range [0, 1] before positional encoding.
Averaging loss over all mask candidates A good candidate mask can be punished because other candidate masks are weaker. Use a SAM-style best-mask loss and train the IoU head to rank masks.
Choosing checkpoint only by validation loss Segmentation quality may improve even when soft loss increases slightly. Track both validation loss and validation IoU. Save best_iou.pt.
Not testing each module separately Shape bugs become hard to debug once the full model is connected. Test image encoder, prompt encoder, mask decoder, full model, and losses using random tensors first.
Ignoring tensor shapes SAM-style models use both sequence tokens and dense feature maps, so shape errors are common. Print and verify every important tensor shape during development.

Most Important Lesson

The most important correction in this project was changing the loss from average multi-mask loss to best-mask training. Since one prompt can have multiple valid interpretations, a SAM-style decoder should be allowed to produce multiple candidate masks.

\[ \text{Bad idea: force all masks to be equally good} \]
\[ \text{Better idea: choose the best mask and train the IoU head to rank candidates} \] 
# Bad: average every candidate mask loss
loss = mean(loss_mask_1, loss_mask_2, loss_mask_3)

# Better: select the best candidate mask
best_idx = argmin(mask_losses)
loss = mask_losses[best_idx] + iou_prediction_loss
