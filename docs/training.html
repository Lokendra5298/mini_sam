Training MiniSAM

This page explains how to train MiniSAM step by step. The goal is simple: teach the model to turn an image plus a prompt into an accurate object mask.

What does training mean?

During training, MiniSAM sees many examples. Each example has an image, a prompt, and the correct mask. The model predicts a mask, compares it with the correct mask, and updates itself to make fewer mistakes.

Step 0: Requirements

Hardware

  • NVIDIA GPU is recommended.
  • 8 GB or more GPU memory is a good starting point.
  • CPU training is possible but much slower.

Software

  • Python 3.8 or newer.
  • PyTorch with CUDA if using GPU.
  • All packages from requirements.txt.

Dataset

The default dataset is Oxford-IIIT Pet. It contains pet images and segmentation masks, so it is perfect for learning object segmentation.

Step 1: Clone and install

First download the repository and install the required packages. 

git clone https://github.com/Lokendra5298/mini_sam.git
cd mini_sam
pip install -r requirements.txt

Then download and prepare the Oxford-IIIT Pet dataset. 

python scripts/download_oxford_pet.py

Step 2: Run quick checks

These tests are useful before training. They check whether each model component can run correctly. 

export PYTHONPATH=$PWD

python tests/test_random_image_encoder.py
python tests/test_random_prompt_encoder.py
python tests/test_random_mask_decoder.py
python tests/test_random_mini_sam.py
python tests/test_random_losses.py
python tests/test_dataset.py
If these tests pass, the model, loss functions, and dataset loader are ready.

Step 3: Understand how prompts are created during training

In a real interactive app, the user gives a click or a box. During training, we already have the correct mask, so we can generate prompts automatically from that mask.

  1. Load image and mask: the mask tells where the object is.
  2. Create a box prompt: find the smallest rectangle that covers the object mask.
  3. Create a point prompt: choose one point inside the object mask.
  4. Train: give image plus prompts to MiniSAM and ask it to predict the mask.

Important: The prompt is not the answer. It is only a hint. The model still has to learn the object boundary.

Step 4: Start training

Use this command for a normal training run. 

python train.py \
  --data_root data/oxford_pet/processed \
  --out_dir runs/my_minisam_run \
  --device_id 0 \
  --epochs 50 \
  --batch_size 32 \
  --workers 8 \
  --lr 1e-4

What each option means

Option Meaning Beginner explanation
--data_root Dataset folder Where the processed images and masks are stored.
--out_dir Output folder Where checkpoints, logs, and results will be saved.
--device_id GPU number Use 0 for the first GPU.
--epochs Number of passes over the dataset 50 means the model sees the full training set 50 times.
--batch_size Images per training step Larger batches can be faster but need more GPU memory.
--workers Data loading workers More workers can load data faster.
--lr Learning rate Controls how strongly the model updates after each mistake.

Step 5: Adjust for your GPU memory

If training stops with an out-of-memory error, reduce the batch size. 

python train.py \
  --data_root data/oxford_pet/processed \
  --out_dir runs/my_minisam_run \
  --device_id 0 \
  --epochs 50 \
  --batch_size 16 \
  --workers 8 \
  --lr 1e-4

If it still fails, try --batch_size 8. Smaller batch size is slower but safer.

Step 6: Read the training logs

During training you may see values such as loss, IoU, and Dice. These tell you whether the model is improving. 

Epoch 1/50
Train Loss: 2.34 | Train IoU: 0.45 | Train Dice: 0.52
Val   Loss: 1.80 | Val   IoU: 0.55 | Val   Dice: 0.65

Epoch 10/50
Train Loss: 0.75 | Train IoU: 0.78 | Train Dice: 0.86
Val   Loss: 0.82 | Val   IoU: 0.75 | Val   Dice: 0.84
Metric What it means Good direction
Loss How wrong the model is. Lower is better.
IoU Overlap between predicted mask and real mask. Higher is better.
Dice Another overlap score, commonly used for segmentation. Higher is better.
Watch validation metrics carefully. If training metrics improve but validation metrics get worse, the model may be overfitting.

Step 7: Evaluate the trained model

After training, evaluate the best checkpoint on the validation set. 

python scripts/evaluate.py \
  --data_root data/oxford_pet/processed \
  --split val \
  --checkpoint runs/my_minisam_run/best_iou.pt \
  --device_id 0 \
  --batch_size 32 \
  --workers 8

This prints final metrics such as IoU, Dice, precision, recall, and pixel accuracy.

Step 8: Visualize predictions

Numbers are useful, but images are easier to understand. This command saves visual examples. 

python scripts/visualize_predictions.py \
  --data_root data/oxford_pet/processed \
  --split val \
  --checkpoint runs/my_minisam_run/best_iou.pt \
  --out_dir runs/my_minisam_run/visuals \
  --device_id 0 \
  --num_samples 32

Check the output folder. You should see the input image, the true mask, the predicted mask, and an overlay.

Common problems and fixes

Problem Likely reason Fix
GPU out of memory Batch size is too large. Use --batch_size 16 or --batch_size 8.
Training is very slow CPU is being used or batch size is very small. Check CUDA installation and use a GPU.
Loss does not decrease Learning rate may be too high or too low. Try --lr 5e-5 or --lr 2e-4.
Dataset error Data path may be wrong. Check --data_root and run the download script again.
ModuleNotFoundError Dependencies are missing. Run pip install -r requirements.txt.

Training checklist

  • Install dependencies.
  • Download and process the dataset.
  • Run component tests.
  • Start with a small batch size if GPU memory is limited.
  • Track validation IoU and Dice.
  • Save and evaluate the best checkpoint.
  • Visualize predictions before trusting the model.
