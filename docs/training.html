Training MiniSAM

This guide shows how to train MiniSAM from scratch. No heavy math - just simple steps!

Step 0: What You Need

Hardware

  • A computer with a GPU (NVIDIA recommended)
  • At least 8GB of GPU memory
  • Or use CPU mode (slower)

Software

  • Python 3.8+
  • PyTorch with CUDA support
  • Other dependencies in requirements.txt

Data

  • Images of objects
  • For each image: a mask showing where the object is
  • Default: Oxford-IIIT Pet dataset (3,680 pet photos)

Step 1: Set Up (Copy-Paste Ready!)

Clone the Project

git clone https://github.com/Lokendra5298/mini_sam.git
cd mini_sam

Install Python Packages

pip install -r requirements.txt

Download Training Data

python scripts/download_oxford_pet.py

This downloads 3,680 pet photos with masks. Takes about 10-30 minutes.

Step 2: Test Everything Works (Optional but Recommended)

Before training, let's verify each part works correctly: 

export PYTHONPATH=$PWD

# Test if image encoder works
python tests/test_random_image_encoder.py

# Test if prompt encoder works
python tests/test_random_prompt_encoder.py

# Test if mask decoder works
python tests/test_random_mask_decoder.py

# Test entire model
python tests/test_random_mini_sam.py

# Test all losses
python tests/test_random_losses.py

# Test if data loads correctly
python tests/test_dataset.py

If all tests pass, you're ready to train! ✅

Step 3: Start Training

Basic Training Command

python train.py \
  --data_root data/oxford_pet/processed \
  --out_dir runs/my_minisam_run \
  --device_id 0 \
  --epochs 50 \
  --batch_size 32 \
  --workers 8 \
  --lr 1e-4

What Do These Options Mean?

Option What It Does Example
data_root Where your training images are data/oxford_pet/processed
out_dir Where to save the trained model runs/my_model
device_id Which GPU to use (0 = first GPU) 0
epochs How many times to go through all data 50
batch_size How many images at once (higher = faster but needs more memory) 32
workers How many processes load data (CPU cores) 8
lr Learning rate (how much model changes per step) 1e-4 (0.0001)

Different Hardware? Adjust batch_size!

If you run out of memory: Reduce batch_size

python train.py \
  --data_root data/oxford_pet/processed \
  --out_dir runs/my_run \
  --batch_size 16

If you want faster training: Increase batch_size

python train.py \
  --data_root data/oxford_pet/processed \
  --out_dir runs/my_run \
  --batch_size 64

Step 4: Watch Training Progress

During training, you'll see output like this: 

Epoch 1/50
Batch 10/100 | Loss: 2.34 | IoU: 0.45 | Dice: 0.52
Batch 20/100 | Loss: 1.98 | IoU: 0.52 | Dice: 0.61
Batch 30/100 | Loss: 1.65 | IoU: 0.58 | Dice: 0.68
...
Validation | IoU: 0.72 | Dice: 0.78

What Do These Numbers Mean?

Metric Meaning Good Value
Loss Error (lower is better) Should decrease over time
IoU Overlap between predicted mask and real mask (0-1, higher is better) 0.7+ is good
Dice Another overlap measure (0-1, higher is better) 0.8+ is good

💡 Tip: Look at validation metrics (at epoch end) not training metrics. Training metrics can be misleading because the model sees training data.

Step 5: Generate Prompts Explained

MiniSAM needs to learn from instructions (clicks and boxes). Here's how the system generates them from mask data:

For Each Training Image:

  1. Load the image and its mask

    The mask shows which pixels are your object (white) and which are background (black).

  2. Create a box prompt

    Find the smallest rectangle that contains all white pixels. That's your box!

  3. Create a point prompt

    Pick a random white pixel. That's your click location!

  4. Train the model

    Give the model the image + box + point, ask it to find the mask.

→

Step 6: Evaluate Your Model

After training, test how well your model performs: 

python scripts/evaluate.py \
  --data_root data/oxford_pet/processed \
  --split val \
  --checkpoint runs/my_minisam_run/best_iou.pt \
  --device_id 0 \
  --batch_size 32 \
  --workers 8

This tests your model on validation data and shows final metrics.

Step 7: Visualize Results

Create pictures showing your model's predictions: 

python scripts/visualize_predictions.py \
  --data_root data/oxford_pet/processed \
  --split val \
  --checkpoint runs/my_minisam_run/best_iou.pt \
  --out_dir runs/my_minisam_run/visuals \
  --device_id 0 \
  --num_samples 32

This creates 32 sample images showing input, ground truth mask, predicted mask, and overlap. Check the output folder to see how well your model did!

Common Issues and Fixes

Problem Cause Solution
GPU out of memory Batch size too large Reduce --batch_size 16 or --batch_size 8
Training very slow Batch size too small or CPU used Increase batch size or use GPU
Loss stays high Model not learning well Try different learning rate: --lr 5e-5
Can't download data Internet or permission issue Download manually from Oxford website
ModuleNotFoundError Dependencies not installed Run pip install -r requirements.txt again

Training Tips

  • ✅ Test first: Run the test scripts before training
  • ✅ Start small: Try with 5 epochs first, then 50
  • ✅ Watch the metrics: Stop if validation loss increases for many epochs
  • ✅ Save checkpoints: The model saves best version automatically
  • ✅ Different data: Try your own images after training on Oxford Pet
  • ✅ Experiment: Try different batch sizes and learning rates

Next: Train Your Own Data!

Once you've trained on Oxford Pet, try with your own images:

  1. Collect images of your object
  2. Create masks (white = object, black = background)
  3. Organize in the same format as Oxford Pet
  4. Run training with your data
  5. See how well it works!
