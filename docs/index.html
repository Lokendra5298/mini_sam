MiniSAM in One Minute

MiniSAM is a small promptable segmentation model inspired by Segment Anything. Instead of directly learning image → mask, it learns:

\[ \text{Image} + \text{Prompt} \rightarrow \text{Object Mask} \]

The prompt can be a point or a box. The model uses the prompt to decide which object in the image should be segmented.

Image
→
Image Encoder
→
Image Features
Point / Box Prompt
→
Prompt Encoder
→
Prompt Tokens
Image Features + Prompt Tokens
→
Two-Way Mask Decoder
→
Final Mask

The key idea is that the mask decoder uses two-way attention to connect the prompt with the correct image region. Then each mask token becomes a dynamic pixel classifier that predicts the foreground object.

Final Validation Performance

MiniSAM was trained from scratch on the Oxford-IIIT Pet segmentation dataset using box and positive point prompts generated from ground-truth masks.

Metric Value
Validation IoU 0.8949
Validation Dice 0.9440
Precision 0.9447
Recall 0.9442
Pixel Accuracy 0.9028
Oracle Best-Mask IoU 0.8954

The selected-mask IoU was very close to the oracle best-mask IoU, showing that the IoU prediction head learned to choose almost the best candidate mask.

Project Goal

The goal is to understand how SAM works internally by implementing a simplified version step by step. Instead of using a pretrained SAM checkpoint, this project builds the main components from scratch and tests every module using random tensors before training on a real segmentation dataset.

What This Project Is and Is Not

What MiniSAM Is

MiniSAM is a small from-scratch implementation of the main SAM-style ideas: image encoder, prompt encoder, two-way mask decoder, mask tokens, and IoU head.

What MiniSAM Is Not

MiniSAM is not a reproduction of Meta's full SAM. It does not use SA-1B-scale training, a huge ViT-H encoder, or large-scale prompt sampling.

Try It Yourself

The project is designed so that each component can be tested independently before training the full model. This makes it easier to understand and debug a SAM-style architecture from scratch.

1. Clone the Repository

git clone https://github.com/Lokendra5298/mini_sam.git
cd mini_sam

2. Install Requirements

pip install -r requirements.txt

3. Download Dataset

python scripts/download_oxford_pet.py

4. Test Each Module

export PYTHONPATH=$PWD

python tests/test_random_image_encoder.py
python tests/test_random_prompt_encoder.py
python tests/test_random_mask_decoder.py
python tests/test_random_mini_sam.py
python tests/test_random_losses.py
python tests/test_dataset.py

5. Train MiniSAM

python train.py \
  --data_root data/oxford_pet/processed \
  --out_dir runs/minisam_oxford_pet_v2_sam_loss_ep50_bs32 \
  --device_id 0 \
  --epochs 50 \
  --batch_size 32 \
  --workers 8 \
  --lr 1e-4

6. Evaluate Final Checkpoint

python scripts/evaluate.py \
  --data_root data/oxford_pet/processed \
  --split val \
  --checkpoint runs/minisam_oxford_pet_v2_sam_loss_ep50_bs32/last.pt \
  --device_id 0 \
  --batch_size 32 \
  --workers 8

7. Visualize Predictions

python scripts/visualize_predictions.py \
  --data_root data/oxford_pet/processed \
  --split val \
  --checkpoint runs/minisam_oxford_pet_v2_sam_loss_ep50_bs32/last.pt \
  --out_dir runs/minisam_oxford_pet_v2_sam_loss_ep50_bs32/visuals_final \
  --device_id 0 \
  --num_samples 32

The recommended development order is: test random tensors first, then test dataset loading, then train, evaluate, and finally visualize predictions.

1. Image Encoder

Converts the input image into dense visual tokens using a ViT-style patch encoder.

[B, 3, 256, 256] → [B, 256, 16, 16]

2. Prompt Encoder

Converts point, box, and mask prompts into sparse and dense prompt embeddings.

point + box → [B, 3, 256]

3. Mask Decoder

Uses two-way attention to connect prompt tokens with image tokens and produce masks.

image + prompt → [B, 3, 256, 256]

4. Losses

Uses focal loss, dice loss, and IoU prediction loss for training.

L = L_mask(best) + L_iou
